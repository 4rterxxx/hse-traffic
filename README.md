# Прогнозирование зарплат на основе резюме hh.ru

Проект для автоматического предсказания зарплат на основе признаков из резюме с сайта hh.ru.

## Структура проекта

```
hse-traffic/
├── pipeline/              # Пайплайн обработки данных
│   ├── features.py       # Извлечение признаков
│   ├── handlers.py       # Обработчики (цепочка ответственности)
│   ├── io_utils.py       # Утилиты ввода/вывода
│   ├── processor.py      # Главный процессор пайплайна
│   └── __init__.py
├── model/                # ML модель
│   ├── linear_reg.py     # Модель линейной регрессии
│   ├── trainer.py        # Обучение модели
│   ├── predictor.py      # Предсказание зарплат
│   └── __init__.py
├── resources/            # Сохраненные веса моделей
├── app.py               # Главный интерфейс приложения
├── requirements.txt     # Зависимости
└── README.md           # Документация
```

## Установка


Установите зависимости:
```bash
pip install -r requirements.txt
```

## Использование

### Задание 1: Пайплайн обработки данных

**Ветка:** `task1-pipeline`

```bash
# Обработка CSV файла с резюме
python app.py process hh.csv
```

**Результат:**
- Создаются файлы `x_data.npy` (признаки) и `y_data.npy` (целевые значения)
- Используется паттерн "Цепочка ответственности" для обработки данных

### Задание 2: Модель предсказания зарплат

**Ветка:** `task2-regression`

```bash
# Обучение модели
python app.py train x_data.npy y_data.npy

# Предсказание зарплат
python app.py predict x_data.npy
```

**Результат:**
- Модель сохраняется в папке `resources/`
- Возвращается список предсказанных зарплат в формате float

## Особенности реализации

### Пайплайн обработки (задание 1)
- **Цепочка ответственности**: последовательная обработка данных через обработчики
- **Извлечение признаков**: парсинг текстовых полей в числовые признаки
- **Обработка ошибок**: устойчивость к некорректным данным

### Модель (задание 2)
- **Улучшенная линейная регрессия**: с регуляризацией и полиномиальными признаками
- **Автоматический подбор гиперпараметров**: поиск лучшей конфигурации
- **Сохранение состояния**: веса модели сохраняются для последующего использования


## Примеры использования

python app.py train x_data.npy y_data.npy # Обучение модели
